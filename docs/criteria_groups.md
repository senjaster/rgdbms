# Список групп критериев

## Группа 1: Установка и обновление
- Критерий 1: Возможность установки СУБД в закрытом контуре
- Критерий 2: Поддержка автоматизированной установки СУБД на операционных системах, входящих в состав ЕРРПО
- Критерий 3: Поддержка автоматизированного обновления между минорными и мажорными версиями

## Группа 2: Сжатие данных
- Критерий 6: Возможность сжатия данных на уровне хранения

## Группа 3: Графическая консоль
- Критерий 10: Наличие графической консоли мониторинга и управления
- Критерий 19: Возможность выполнения запросов к СУБД непосредственно из графической консоли
- Критерий 20: Возможность графического представления планов выполнения запросов
- Критерий 21: Возможность просмотра информации о структуре и размерах БД, объектах БД

## Группа 4: Инструментарий анализа производительности
- Критерий 4: Инструментарий для анализа производительности
- Критерий 15: Наличие агента для сбора информации о сервере СУБД
- Критерий 12: Наличие преднастроенных метрик в графической консоли
- Критерий 13: Возможность добавления собственных метрик
- Критерий 14: Просмотр исторических данных по объектам мониторинга

## Группа 5: Интеграция с LDAP
- Критерий 25: Возможность использования внешнего провайдера аутентификации
- Критерий 18: Возможность интеграции графической консоли с внешней системой аутентификации
- Критерий 27: Настройка ролевой модели доступа на основе групп, атрибутов пользователя с использованием внешнего провайдера (LDAP)

## Группа 6: Шифрование
- Критерий 24: Возможность использования корпоративного TLS/SSL сертификата для компонентов решения
- Критерий 35: Наличие инструментов прозрачного шифрования данных на уровне хранения

## Группа 7: Парольные политики
- Критерий 28: Наличие встроенного механизма парольных политик для управления требованиями к сложности паролей
- Критерий 29: Наличие встроенного механизма парольных политик для управления жизненным циклом паролей

## Группа 8: Аудит
- Критерий 32: Наличие встроенного инструмента системного аудита действий пользователей, в том числе администраторов
- Критерий 34: Возможность хранения журнала аудита в выделенном файле

## Группа 9: Управление доступом
- Критерий 36: Поддержка дискреционной (DAC) модели управления доступом
- Критерий 37: Поддержка ролевой (RBAC) модели управления доступом

## Группа 10: Отказоустойчивость и кластеризация
- Критерий 44: Наличие в поставке инструментов для построения отказоустойчивых конфигураций
- Критерий 53: Встроенный отказоустойчивый кластер (без использования сторонних компонентов)
- Критерий 60: Возможность построения active-active кластера
- Критерий 56: Возможность проксирования запросов на активный узел
- Критерий 45: Автоматическое переключение на резервный узел
- Критерий 57: Поддержка конфигурации кластера с несколькими мастерами

## Группа 11: Резервное копирование
- Критерий 48: Наличие в поставке инструментов, позволяющих выполнять резервное копирование и восстановление
- Критерий 51: Наличие в поставке инструментов, позволяющих выполнять резервное копирование и восстановление на S3
- Критерий 52: Восстановление из резервной копии отдельной БД

## Группа 12: Параллельное выполнение запросов
- Критерий 58: Вертикально: параллельные планы выполнения
- Критерий 63: Распределенное выполнение запроса: запрос может выполняться параллельно с использованием CPU на разных узлах
- Критерий 59: Вертикально: учет NUMA-нод при аллокации памяти

## Группа 13: Распределенные транзакции и балансировка
- Критерий 61: Поддержка многоузловых транзакций
- Критерий 55: Наличие механизма балансировки читающей нагрузки
- Критерий 64: Наличие механизма автоматического распределения данных по узлам

## Группа 14: Compute-storage separation и масштабирование
- Критерий 67: Compute-storage separation: наличие отдельных слоев, отвечающих за обработку запросов и за хранение данных
- Критерий 69: Наличие возможности добавить узлы, отвечающие только за выполнение запросов
- Критерий 70: Эластичность: наличие механизмов, позволяющих динамически добавлять и исключать узлы из кластера в зависимости от нагрузки
- Критерий 68: Наличие возможности расширять существующий кластер
- Критерий 71: Ограничения на практически возможный размер кластера

## Группа 15: Индексы и секционирование
- Критерий 76: Поддержка глобальных индексов для секционированных таблиц
- Критерий 77: Автосекционирование (автоматическое создание секций)

## Группа 16: Работа с Parquet и S3
- Критерий 80: Возможность выгрузки данных штатными средствами из СУБД в parquet файлы
- Критерий 81: Возможность обращения штатными средствами СУБД к parquet файлам для выполнения аналитических запросов
- Критерий 82: Возможность использования в одном запросе данных, хранимых в СУБД и parquet файлах

## Группа 17: Очереди и Change Data Capture
- Критерий 84: Расширенное управление очередями (Advanced Queueing)

## Группа 18: Управление ресурсами
- Критерий 7: Возможность квотирования или приоритезации ресурсов (cpu, ram, disk)
